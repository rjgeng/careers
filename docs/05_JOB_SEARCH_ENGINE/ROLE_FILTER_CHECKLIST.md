# Role Filter Checklist — Age‑Aware (LinkedIn / Indeed)

**Purpose:** Quickly screen job postings to avoid age‑biased roles and focus time on positions with the highest probability of success.

Use this as a **binary filter**. If too many ❌ appear, skip the role.

---

## 1️⃣ Job Title Filter (first 10 seconds)

### Strong titles ✅

* Software Engineer – **Platform / Internal Tools**
* Frontend Engineer – **Enterprise / Admin / Dashboard**
* UI Engineer – **Design System / Tooling**
* Software Engineer – **Modernization / Migration**
* Frontend Engineer – **AI‑Enabled Product** (non‑ML)

### Weak titles ❌

* Rockstar / Ninja / Hacker
* Growth Engineer
* Web3 / Social / Consumer App (early‑stage)
* Prompt Engineer (frontend‑only)

---

## 2️⃣ Language Scan (copy‑paste skim)

### Green‑flag phrases ✅

* "internal tools"
* "platform UI"
* "admin dashboard"
* "enterprise customers"
* "design system"
* "maintainability"
* "migration" / "modernization"
* "compliance" / "reliability"

### Red‑flag phrases ❌

* "fast‑paced startup"
* "move fast and break things"
* "wear many hats"
* "high‑energy environment"
* "recent graduates"
* "growth mindset" (without context)

---

## 3️⃣ Responsibility Check (core signal)

### Favor roles where responsibilities include ✅

* maintaining existing systems
* improving reliability
* refactoring / migrating legacy code
* building internal dashboards
* collaborating with non‑engineering stakeholders

### Avoid roles emphasizing ❌

* rapid greenfield iteration only
* viral growth metrics
* constant redesign
* undefined ownership

---

## 4️⃣ AI Mentions (important nuance)

### Healthy AI framing ✅

* "AI‑enabled workflows"
* "developer productivity"
* "tooling / platform"
* "AI safety / reliability" (frontend)

### Unhealthy AI framing ❌

* "AI will design the system"
* "prompt‑only development"
* "no‑code / one‑click SaaS"

---

## 5️⃣ Experience Requirements

### Good signs ✅

* experience range (e.g., "5+ years" without upper bound)
* emphasis on impact, not age
* no graduation year mentioned

### Warning signs ❌

* "0–3 years" hard limits
* "recent graduate" emphasis
* heavy focus on "learning speed"

---

## 6️⃣ Interview Style Clues

### Likely structured & fair ✅

* mentions system design discussions
* mentions collaboration / documentation
* mentions trade‑offs

### Likely biased / chaotic ❌

* trivia‑heavy interview descriptions
* "culture fit" without definition
* vague expectations

---

## 7️⃣ Quick Scoring (final decision)

* 0–2 ❌ → **Apply**
* 3–4 ❌ → **Only apply if referral**
* 5+ ❌ → **Skip**

---

## 8️⃣ Resume Framing Reminder (before applying)

Use this language:

> "I use AI‑assisted tooling to accelerate UI work while retaining architectural ownership and system boundaries."

Avoid:

* age indicators
* graduation years
* course‑only descriptions

---

## Bottom Line

> **Don’t fight the market.
> Route around bias using role selection.**

This checklist protects your time, confidence, and leverage.
